How the platform works.

A working reference for the Kasagadi AI fact-checking marketplace — actors, claim lifecycle, authentication, storage, codebase layout, and the conventions that hold it all together.

Read the full manual → View as markdown
Kasagadi AI is a Ghana-focused platform where members submit claims they want investigated, admins triage and assign accredited fact checkers, and verdicts are published to a public marketplace. This documentation explains how each role moves through that flow and where each piece lives in the codebase.
Stack
Ruby 3.3 · Rails 8 · PostgreSQL
Frontend
Tailwind 4 · Stimulus · Importmap
Storage
Active Storage on Cloudflare R2
Tests
~190 passing in <5s
The roles

Three actors keep the platform moving

Member
Self-signs up, drafts claims, attaches evidence, submits for review.
Fact checker
Invited by an admin, picks up assignments, researches, submits a verdict.
Admin
Triages incoming claims, assigns checkers, approves verdicts, manages invitations.
